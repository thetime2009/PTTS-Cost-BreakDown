# กฎการทำงานสำหรับโปรเจกต์นี้ (form.html)

## กฎตายตัว — ห้ามลืม

ทุกครั้งที่แก้โค้ดใน `form.html` เสร็จในแต่ละรอบ ต้องทำ 2 อย่างนี้พร้อมกันเสมอ:

1. **อัปเดตเลขเวอร์ชัน** — ที่ `<span class="ph-chip ver">vX.X</span>` (หัวกระดาษ "PTS Cost Breakdown")
2. **เพิ่มรายการใน Dev Logs** — เพิ่ม entry ใหม่ไว้บนสุดของ array `DEV_LOGS` (ใกล้ท้ายไฟล์ ก่อน `</body>`) โดยใส่:
   - `ref`: เลขอ้างอิง รูปแบบ `#REV-YYYYMMDD-X` (X = A, B, C, ... ไล่ตามลำดับของวันนั้น)
   - `date`: วันที่ปัจจุบัน
   - `current: true` (และลบ/เปลี่ยน `current: true` ของ entry ก่อนหน้าออก)
   - `items`: รายการสิ่งที่แก้ไขสั้นๆ เป็นข้อๆ (ภาษาไทย)

## กฎอื่นๆ จากผู้ใช้

- ถ้าต้อง Redeploy Apps Script (Code.gs) ต้องแจ้งผู้ใช้ทุกครั้ง
- เรื่องการอัปขึ้น GitHub ผู้ใช้จะทำเอง ไม่ต้องทำให้ จนกว่าจะสั่ง
